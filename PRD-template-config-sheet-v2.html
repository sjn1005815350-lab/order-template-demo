3、需求概述

3.1、需求背景

当前报单排序配置需要同时解决“配置复用”和“客户最终明细落地”两个问题。新版方案保留顺序配置和模板配置的复用能力，但客户配置保存时落到客户维度最终明细。模板仅作为新增客户配置时的快速填入工具，不作为客户最终保存字段。

3.2、需求目标

  • 维护不同对冲渠道的报单顺序，并通过“顺序名称”复用。
  • 维护模板配置，沉淀常用业务类型、合约类型、交易市场、资产品种、交易方向和顺序名称组合。
  • 新增客户配置时选择模板名称快速填入模板下明细。
  • 客户配置最终保存交易账户维度明细数据，不保存模板引用关系。

3.3、需求范围

范围内：BCT 报单排序模板配置页面、顺序配置、模板配置、客户配置、新增客户配置弹框、模板快速填入、客户最终明细保存。

范围外：对冲渠道基础资料维护、交易账户基础资料维护、模板审批发布、批量导入导出、智能路由执行链路改造、模板变更自动同步客户配置。

3.5、阶段实现目标

一阶段完成新版页面、顺序配置、模板配置、客户配置新增弹框和最终明细保存。二阶段可扩展客户配置编辑删除、批量导入导出、按模板刷新客户配置、配置审批和版本追溯。

术语说明

3.7、领域知识

智能路由报单时通常根据交易账户、业务类型、合约类型、交易市场、资产品种、交易方向等维度匹配顺序名称，再按该顺序名称对应的渠道优先级报单。

3.8、业务流程

进入页面 → 维护顺序配置 → 维护模板配置 → 点击客户配置新增 → 选择交易账户 → 可选模板名称快速填入 → 调整明细 → 保存客户最终明细

4、功能设计

BCT系统模块

4.1、设计功能点一：模板基础配置 - 顺序配置

顺序配置用于维护不同对冲渠道的报单先后顺序。每个顺序配置包含顺序名称、备注和多条对冲渠道明细。

位置序号名称类型变更说明只读校验校验报错信息默认值枚举值备注
顺序配置母表1顺序名称文本/输入优化否必填、建议唯一请输入顺序名称空-供后续引用
顺序配置子表2报单排序数字优化是系统计算无1-根据行顺序自动生成
顺序配置子表3对冲渠道下拉/文本优化否必填请选择对冲渠道空QFII2、Mock、CMS_QFII、CMS_SC、HTSC、CICC、IB渠道基础数据

4.2、设计功能点二：模板基础配置 - 模板配置

模板配置用于维护可复用的模板明细。模板用于客户配置新增时快速填入，不作为客户最终保存字段。

位置序号事件校验校验报错信息影响备注
模板配置母表1点击操作列“编辑”无无当前模板母表行进入编辑态，模板名称、备注可修改，按钮变为“保存”编辑期间不影响已保存客户配置
模板配置母表2点击操作列“保存”模板名称必填、建议唯一请输入模板名称/模板名称已存在保存模板母表字段，退出编辑态刷新新增客户配置弹框模板名称下拉
模板配置母表3点击操作列“删除”删除前确认当前模板存在明细，请确认是否删除删除模板及其模板明细不影响已通过该模板保存的客户配置明细
模板配置子表4点击操作列“编辑”无无当前明细行进入编辑态，全部明细字段可修改-
模板配置子表5点击操作列“保存”明细必填字段完整、同一模板下业务维度建议唯一请完善模板明细/当前模板下存在重复明细保存当前明细并退出编辑态唯一性建议按业务类型+合约类型+交易市场+资产品种+交易方向
模板配置子表6点击操作列“删除”删除需确认无删除当前模板明细不影响已保存客户配置

功能逻辑：模板名称只在新增客户配置时用于快速填入；点击编辑后仅当前行进入编辑态；点击保存时进行必填和唯一性校验；删除模板或模板明细只影响后续快速填入，不影响已保存客户配置。

客户配置列表展示交易账户维度最终保存的智能路由明细。新增客户配置弹框写入本功能点的字段与交互中，不单独作为功能点。

用户故事/验收条件

作为运营人员，我希望通过模板快速新增客户配置明细，以便减少重复录入并保证最终配置可核对。

Given 系统已存在模板配置和顺序配置
When 用户在新增客户配置弹框中选择交易账户并选择模板名称
Then 系统应将模板明细填入弹框明细表
And 弹框明细表不展示交易账户列
When 用户点击保存
Then 系统应保存交易账户维度最终明细
And 客户配置列表不展示模板名称字段

作为交易运营人员，我希望在模板填入后继续调整客户明细，以便处理客户个性化路由要求。

Given 用户已通过模板名称快速填入客户配置明细
When 用户删除部分明细并新增个性化明细后点击保存
Then 系统应保存用户调整后的最终明细
And 不应强制保存模板原始完整明细

5、非功能性要求(或有)
